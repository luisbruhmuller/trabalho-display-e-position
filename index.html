Display e Position

Exemplos visuais e praticos das propriedades

Display

Display 1: block

ele faz com que um elemento HTML ocupe toda a largura disponível, elementos de bloco se alinham verticalmente, e é permitido regular largura (width) e altura (height) além de margem e preenchimentos (padding). e ele inicia uma nova linha empurrando o próximo elemento para baixo.

Item 1 (block) Item 2 (block)

Display 2: inline

ele faz com que elementos em HTML fluam na mesma linha, minimizando o tamanho ocupado usando apenas o necessário, sem iniciar uma nova linha, Elementos feitos com Inline não respeitam propriedades de altura e largura e suas margens superiores e inferiores não funcionam, inline é ideal para estruturar texto e itens lado a lado.

Item 1 (inline)
Item 2 (inline)

Display 3: inline-block

Ele combina o melhor dos dois mundos do inline e do Block, ele permite que elementos fiquem lado a lado na mesma linha (em forma de texto), e ele aceita propriedades de tamanho (height e width), o que antes não seria possível com o inline, seria ideal para criar menus, botões e cards.

Botão 1
Botão 2

Display 4: flex

É uma propriedade CSS3 que transforma um container HTML em um contêiner flexível, alinhando automaticamente seus elementos filhos (itens) em uma linha ou coluna, é a forma mais eficiente de criar layouts responsivos, facilitando a distribuição de espaço.

Flex 2
Flex 3

Display 5: inline-flex

Ela combina o comportamento do inline (que já foi explicado anteriormente) e dp Flex, que cuida dos elementos internos (filhos), e ele não força uma quebra de linha no container.

Texto antes A B Texto depois

Display 6: grid

É uma ferramenta CSS que cria layouts bidimensionais (linhas e colunas simultaneamente), para organizar elementos em uma página web. ele é mais eficiente do que outros como flex, pois ele é muito preciso.

Display 7: inline-grid

é uma propriedade que define um elemento como container de grade (grid), permitindo o uso de linhas e colunas criando um tipo de tabela, e ele usa apenas o espaço necessário permitindo elementos ao seu lado.

Texto G1 G2 ao lado

Display 8: flow-root

no CSS cria um novo tipo de formação de Bloco (BFC) forçando o container a envolver elementos float (flutuantes), sem precisar de técnicas antigas como clearfix, e assim acaba protegendo o conteúdo e não tendo colapso de margens.

Float

O container "flow-root" engloba o flutuante sem colapsar.

Display 9: table

Ele funciona fazendo que elementos HTML como <div> virarem tabelas <table>, organizando conteúdo em linhas e células sem usar tags de tabela estruturais. é ideal para alinhamentos verticais complexos e layouts responsivos, definindo comportamentos como table-row e table-cell.

define que um elemento ocupe toda a largura possível, iniciando em uma nova linha e forçando a quebra de linha.

ele faz com que elementos fiquem em blocos verticais, ocupando toda a largura possível, e flex faz com que eles fiquem alinhados de forma responsiva.
